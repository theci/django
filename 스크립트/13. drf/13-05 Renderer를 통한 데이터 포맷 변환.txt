이 글에서는 **Django REST Framework (DRF)**에서 **렌더러(Renderer)**를 사용하는 방법과 커스텀 렌더러를 구현하는 방법에 대해 설명합니다. 
렌더러는 파이썬 데이터를 특정 포맷으로 변환하여 응답하는 역할을 합니다.

1. 렌더러 기본 개념
렌더러는 Django REST Framework에서 시리얼라이저를 통해 변환된 파이썬 데이터(기본 데이터 타입)를 클라이언트가 요청한 포맷으로 변환하여 반환하는 역할을 합니다.
기본적으로 JSON 렌더러와 Browsable API 렌더러가 제공됩니다.
JSON 렌더러: 응답을 JSON 형식으로 변환합니다.
Browsable API 렌더러: HTML 형식으로 변환하여 API를 웹 브라우저에서 쉽게 테스트할 수 있게 합니다.
렌더러는 요청된 포맷에 맞춰 자동으로 선택됩니다. 예를 들어, URL의 포맷 인자나 요청 헤더의 Content-Type을 기반으로 어떤 렌더러를 사용할지 결정합니다.

2. 렌더러 결정 방식
DRF는 요청된 포맷 인자(예: .json 등)와 Content-Type 헤더를 바탕으로 적합한 렌더러를 선택합니다.
브라우저에서 요청을 보낼 경우, Browsable API 렌더러가 선택되고, .json으로 포맷을 지정하면 JSON 렌더러로 응답합니다.
지원되지 않는 포맷을 요청할 경우, 404 응답을 받게 됩니다.

3. 커스텀 렌더러 구현
렌더러 클래스는 BaseRenderer 클래스를 상속받아 구현합니다.
필수 메서드인 render 메서드를 오버라이드하여 응답 데이터를 특정 포맷으로 변환합니다.
미디어 타입은 응답 헤더에서 사용될 컨텐츠 타입(예: application/json, application/xml)을 설정합니다.
예시:
엑셀 파일 렌더러: pandas 라이브러리를 사용하여 엑셀 파일을 생성하는 렌더러를 구현할 수 있습니다.
워드 클라우드 렌더러: wordcloud 라이브러리를 사용하여 워드 클라우드를 생성하고, 이를 이미지 포맷(SVG 등)으로 반환하는 렌더러입니다.

4. 엑셀 파일 렌더러
pandas의 ExcelWriter를 사용하여 엑셀 파일로 데이터를 변환합니다. 이때 엑셀 파일의 각 컬럼을 잘 처리해야 하며, 중첩된 데이터는 **json_normalize**를 통해 평탄화하여 엑셀 파일로 변환할 수 있습니다.

5. 워드 클라우드 렌더러
wordcloud 라이브러리를 사용하여 텍스트 데이터를 기반으로 워드 클라우드를 생성합니다. 워드 클라우드는 **벡터 이미지 포맷(SVG)**으로 생성하여 텍스트가 확대되더라도 선명도가 유지됩니다.

6. 렌더러 설정
프로젝트 전역 설정: 프로젝트 전체에서 사용할 렌더러를 설정하려면 settings.py 파일의 REST_FRAMEWORK 설정에 DEFAULT_RENDERER_CLASSES 키를 사용하여 활성화합니다. 예시:
python
코드 복사
REST_FRAMEWORK = {
    'DEFAULT_RENDERER_CLASSES': (
        'rest_framework.renderers.JSONRenderer',
        'myproject.renderers.ExcelRenderer',
        'myproject.renderers.WordCloudRenderer',
    ),
}
개별 API 뷰 설정: 각 뷰에서 직접 사용할 렌더러를 설정할 수도 있습니다. 이 방법은 특정 API 뷰에서만 별도의 렌더러를 적용하고자 할 때 유용합니다.

7. 응답 예시
기본 응답: 웹브라우저를 통해 요청하면 Browsable API 렌더러로 HTML 형태로 응답을 받습니다.
JSON 응답: .json 포맷을 요청하면 JSON 렌더러가 데이터를 JSON 형식으로 반환합니다.
엑셀 응답: .xls 포맷을 요청하면 엑셀 파일로 데이터를 응답받을 수 있습니다.
워드 클라우드 응답: .svg 포맷을 요청하면 워드 클라우드를 SVG 포맷으로 반환합니다.

8. 워드 클라우드와 엑셀 파일 개선
엑셀 파일: 중첩된 데이터는 pandas.json_normalize() 함수를 사용하여 평탄화하여 엑셀에 저장할 수 있습니다.
워드 클라우드: 컬럼명이나 숫자 데이터를 제외하고 텍스트 데이터만을 활용하여 보다 의미 있는 워드 클라우드를 생성할 수 있습니다. 이때, .id로 끝나는 컬럼을 제외하여 워드 클라우드를 개선합니다.
