이번 내용은 CSS 프레임워크를 사용하여 웹 페이지의 스타일을 적용하는 방법에 대해 다루고 있습니다. 주요 내용은 Bootstrap 5와 Tailwind CSS 두 가지 프레임워크를 사용하여 스타일링을 적용하는 과정입니다. 아래에 요약을 정리해드립니다.

1. Bootstrap 5 사용
Bootstrap 5는 미리 정의된 스타일과 클래스를 제공하여 빠르게 웹 페이지의 디자인을 적용할 수 있는 CSS 프레임워크입니다.
적용 방법:
Bootstrap CSS 파일을 웹 페이지에 포함시키기 위해, CDN 링크를 <head> 태그 안에 추가합니다.
Bootstrap 클래스명을 HTML 요소에 추가하면, 미리 설정된 스타일이 자동으로 적용됩니다.
예시:
container 클래스를 사용하면 페이지 콘텐츠가 가운데 정렬됩니다.
fw-bold 클래스로 글자를 볼드 처리합니다.
form-control 클래스를 사용하여 입력 폼에 기본 스타일을 적용합니다.
table, table-hover, table-bordered 클래스를 사용하여 테이블에 스타일을 적용하고, 마우스를 올리면 호버 효과가 추가됩니다.
장점: 빠르게 스타일을 적용할 수 있으며, 다양한 UI 컴포넌트를 미리 사용할 수 있습니다.
2. Tailwind CSS 사용
Tailwind CSS는 유틸리티 클래스 기반의 CSS 프레임워크로, 개별적인 스타일을 클래스를 통해 조합하여 적용합니다. 자유도는 높지만, 스타일을 하나하나 정의해야 하므로 번거로울 수 있습니다.
적용 방법:
Tailwind CSS를 적용하려면 CDN 링크를 <head> 태그에 추가합니다.
여러 유틸리티 클래스를 HTML 요소에 추가하여 스타일을 적용합니다.
예시:
mx-auto로 콘텐츠를 가운데 정렬합니다.
font-bold로 볼드체를 적용하고, text-xl로 글자 크기를 설정합니다.
w-full, p-2, border, rounded 등으로 입력 폼의 크기, 여백, 테두리 등을 설정합니다.
table, divide-y, border 클래스를 사용하여 테이블에 스타일을 적용합니다.
장점: 자유도가 높고, UI 컴포넌트를 재활용하는 방식에서 강점을 보입니다. 세밀한 스타일 조정이 가능하지만 코드가 길어질 수 있습니다.
3. Bootstrap 5 vs Tailwind CSS
Bootstrap 5는 미리 정의된 스타일을 빠르게 적용할 수 있어, 기본적인 웹 페이지 디자인을 빠르게 완성할 수 있습니다. 그러나 스타일이 고정되어 있어 맞춤형 디자인을 만들 때는 제한이 있을 수 있습니다.
Tailwind CSS는 유틸리티 클래스를 조합하여 사용하며, 디자인의 자유도가 높습니다. 그러나 각 스타일을 하나씩 설정해야 하므로 처음에는 조금 번거로울 수 있습니다. 주로 UI 컴포넌트 개발이나 커스터마이징에 유리합니다.

핵심 요약:
Bootstrap 5:
미리 정의된 스타일로 빠르게 디자인을 적용.
UI 컴포넌트를 빠르게 사용할 수 있어 간편하게 스타일링 가능.
간단한 웹 페이지에 적합.

Tailwind CSS:
유틸리티 기반으로 자유도가 높음.
세밀한 스타일링과 커스터마이징 가능.
UI 컴포넌트 재활용과 맞춤형 디자인에 유리.
