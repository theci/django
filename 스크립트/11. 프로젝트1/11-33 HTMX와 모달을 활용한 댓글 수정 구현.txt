구현해 보겠습니다 수정하기 버튼을 클릭하면 modal 창으로 수정 폼이 뜨고 저장하면
modal 폼이 닫히고 수정 성공 메시지가 toast 메시지도 뜨고 댓글 목록도 자동으로
갱싱될 것입니다
언더바 코멘트 리스트 점 HTML 템플릿을 수정하여 수정하기 버튼을 추가하겠고요.
이후에 삭제하기 버튼도 옆에 추가할 것이기에 플렉스 레이아웃을 횡으로 적용하겠습니다.
포트폴리오 앱의 comment-edit 이름의 뷰를 구현하겠고요.
url-captured-value 인자로 노트의 pk 기본키와 수정할 댓글의 기본키를
전달받겠습니다.
update-view 클래스를 상속받고요.
comment-create-view 구현과 거의 유사하게 구현할 수 있습니다.
model, form-class, template-name 속성을 그대로 복사하고요.
login-required, hx 장식자도 복사해서 적용해줍니다.
댓글은 댓글 작성자만 수정할 수 있도록 get_queryset 메서드를 재정의해서 author
작성자 필드에 대한 조회 조건으로 현재 요청 유저를 적용하겠습니다
그리고 CommentForm 클래스에서는 request 이름의 인자가 추가로 필요하니깐요.
get_form_kwargs 메서드도 복사해서 적용합니다.
유효성 검사 후에 모델을 통해 데이터베이스에 적용하는 FormValid 메서드도 그대로
복사하겠고요. 댓글 생성 단계에서 이미 작성자 필드와 노트 필드는 이미 지정되어 저장이
되어 있죠. 그러니 댓글 수정 단계에서는 지정할 필요가 없으니 제거합니다.
댓글 수정하기 버튼에 링크를 추가하고 브라우저 캐싱을 원천적으로 막도록 hxgetwith
타임스탬프 속성도 추가하고요.

클릭하면 수정 폼이 뜨고, 내용 수정하시고 저장 누르시면 저장이 되고, 못할 폼이 자동으로 닫히고, 성공 토스트 메세지도 뜨고, 댓글 목록도 자동으로 갱신되었습니다.
