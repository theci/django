특정 필드만 내가 업데이트 하겠다고 하신다면 여기에 updateFields라는 속성으로
likeCount 이렇게만 지정하시면 해당 필드로만 업데이트 코리가 수행이 되는 거야 다시
페이지 이동한 다음에 몇 곡을 선택을 하고 그리고 업데이트해서 확인을 해보시면 이번에는
코디가 좀 더 심플해졌죠
만약에 어떤 포스팅의 내용 같은 것들이 큰 대상에 대해서 컨텐츠 내용이 바뀌지 않았음에도
.save 호출을 통해서 매번 컨텐츠 내용을 업데이트를 하게 되면 데이터베이스와의 불필요한
트래픽이 발생하는 거죠.
상황에 따라 이렇게 업데이트 필드를 지정하시거나 혹은 벌크 업데이트를 활용하셔서 갱신을
원하는 필드만 적절히 호출하는 것이 필요할 수 있습니다.




이번 내용은 Django 디버그 툴바(Django Debug Toolbar)를 설치하고, 이를 통해 Django 프로젝트에서 뷰 처리 및 데이터베이스 쿼리 등의 디버깅 정보를 추적하는 방법을 설명한 것입니다. 주요 내용은 다음과 같습니다:

1. Django 디버그 툴바 설치 및 설정
설치: django-debug-toolbar를 설치합니다. requirements.txt에 해당 라이브러리를 추가하여 설치할 수 있습니다.
앱 등록: INSTALLED_APPS에 debug_toolbar를 추가합니다. 이 설정은 개발 서버에서만 활성화되도록 해야 합니다. 즉, DEBUG = True일 때만 활성화됩니다.
미들웨어 등록: MIDDLEWARE 설정에서 debug_toolbar.middleware.DebugToolbarMiddleware를 가장 먼저 추가해야 합니다. 이는 요청 처리 시에 가장 먼저 호출되고, 응답 시에 가장 마지막에 호출되기 때문에 디버그 정보를 가장 많이 수집할 수 있습니다.
Internal IPs 설정: 디버그 툴바가 보여질 IP를 제한할 수 있습니다. 기본적으로 127.0.0.1로 설정하여 개발 서버에서만 디버그 툴바를 확인할 수 있게 합니다.
URL 패턴 등록: 프로젝트의 urls.py 파일에서 debug_toolbar.urls를 포함시켜야 합니다. 이는 디버그 툴바 관련 URL을 설정하는 부분입니다.
2. Django 디버그 툴바 활용
디버그 툴바가 활성화되면, 페이지 하단에 다양한 디버깅 정보를 제공하는 패널이 나타납니다. 이를 통해 각종 정보를 실시간으로 확인할 수 있습니다.
SQL 패널: 수행된 SQL 쿼리 및 관련 정보를 보여주는 패널로, 쿼리의 실행 시간, 쿼리 내용 등을 확인할 수 있습니다. 이는 데이터베이스 최적화에 큰 도움이 됩니다.
뷰 함수 및 템플릿 처리: Django 디버그 툴바는 뷰 함수의 처리 흐름, 사용된 템플릿, 그에 따른 변수 값 등을 쉽게 확인할 수 있도록 도와줍니다.
3. SQL 쿼리 추적
실행된 SQL 쿼리: 페이지에서 수행된 SQL 쿼리를 디버그 툴바의 SQL 패널을 통해 확인할 수 있습니다. 예를 들어, 특정 액션(좋아요 갱신)을 실행했을 때 어떤 쿼리가 실행되었는지 파악할 수 있습니다.
리다이렉트 처리: 액션 후 페이지 리다이렉션이 발생할 때, Intercept Redirect 옵션을 활성화하면 페이지 이동을 끊고 쿼리 결과를 바로 확인할 수 있습니다. 이를 통해 리다이렉트 후 SQL 쿼리 내역도 확인할 수 있습니다.
4. 쿼리 최적화 예시
벌크 업데이트와 .save 사용 비교: bulk_update와 .save()를 비교하여 실행되는 쿼리의 차이를 확인할 수 있습니다. 예를 들어, .save()를 호출할 경우 모든 필드에 대해 업데이트가 발생하지만, bulk_update를 사용하면 특정 필드만 업데이트하는 쿼리를 생성할 수 있습니다. 이를 통해 불필요한 데이터베이스 트래픽을 줄일 수 있습니다.
5. 템플릿 최적화
HTML 구조: Django 디버그 툴바는 body 태그의 끝에 HTML, CSS, JavaScript 코드를 삽입하여 동작합니다. 따라서 기본 HTML 구조에서 body 태그가 끝나는 부분에 디버그 툴바가 정상적으로 동작할 수 있는지 확인해야 합니다.
템플릿 작성 시 주의사항: Django 디버그 툴바가 제대로 작동하려면 body 태그를 포함한 기본적인 HTML 구조가 필요합니다. 템플릿 개발 시 기본 HTML5 문법을 준수하는 것이 중요합니다.
6. 정리
Django 디버그 툴바는 개발 서버에서 뷰 처리, SQL 쿼리, 템플릿 렌더링 과정 등을 쉽게 디버깅할 수 있도록 도와주는 필수적인 도구입니다. 이를 활용하면 성능 최적화와 문제 해결에 많은 도움이 될 수 있습니다. 디버그 툴바에서 제공하는 SQL 패널, 뷰 패널, 템플릿 패널 등을 통해 코드의 실행 흐름과 쿼리 최적화 상태를 실시간으로 추적할 수 있습니다.

Django 디버그 툴바는 개발 중에 매우 유용한 도구이지만, 실 서비스에서는 보안상 정보를 노출할 수 있으므로 사용을 자제해야 합니다.
