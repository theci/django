이번 실습은 로컬 환경에서 같은 루트 도메인을 사용하여 CORS와 세션 쿠키 공유 문제를 해결하는 과정입니다. 도메인 설정을 통해 로컬 환경에서 도메인 기반 테스트를 진행할 수 있으며, 이를 통해 Django와 프론트엔드 서버 간의 통신을 원활히 처리할 수 있게 됩니다.

주요 내용은 다음과 같습니다:

1. 로컬 환경에서 도메인 설정
목표: 로컬 개발 환경에서 도메인을 설정하여 테스트 환경을 만들기.
호스트 파일 수정:
Windows: C:\Windows\System32\drivers\etc\hosts 파일을 수정하여, 127.0.0.1 IP 주소에 mydj.com과 api.mydj.com 도메인을 추가합니다.
Mac/Linux: /etc/hosts 파일을 수정하여 동일한 IP 주소와 도메인을 추가합니다.
효과: 이 설정은 로컬에서만 적용되며, mydj.com과 api.mydj.com이 모두 127.0.0.1을 가리키게 됩니다.
2. Django 서버 설정
Django 서버에서 도메인 설정:
settings.py에서 **ALLOWED_HOSTS**에 api.mydj.com을 추가하여 해당 도메인에서 오는 요청을 허용합니다.
환경 변수에서 도메인을 받아오도록 .env 파일에 도메인 값을 추가하고, Django 서버를 재시작합니다.
3. CORS 설정
문제 발생: 같은 루트 도메인을 사용하더라도, 서브도메인(예: api.mydj.com과 mydj.com)은 다른 출처로 간주됩니다. 이로 인해 CORS 오류가 발생합니다.
CORS 설정:
settings.py의 **CORS_ALLOWED_ORIGINS**에 http://mydj.com:3000(프론트엔드 서버 주소)을 추가하여, 프론트엔드에서 Django 서버로의 HTTP 요청을 허용합니다.
**CORS_ALLOW_CREDENTIALS**를 True로 설정하여 쿠키 자동 전송을 허용합니다.
4. Session Cookie Domain 설정
쿠키 공유 문제 해결: api.mydj.com과 mydj.com 간의 세션 쿠키 공유를 위해 Session Cookie Domain을 설정합니다.
설정 방법:
SESSION_COOKIE_DOMAIN을 ".mydj.com"으로 설정하여, 루트 도메인(mydj.com) 하위의 모든 서브도메인에서 쿠키를 공유할 수 있도록 합니다.
효과: 이제 api.mydj.com과 mydj.com 모두 같은 세션 쿠키를 공유할 수 있습니다.
5. 프론트엔드 설정 (Fetch API)
CORS 오류 발생: 처음에는 프론트엔드에서 localhost:3000으로 요청을 보내고 있었으므로, CORS 정책 위반이 발생합니다.
수정: 요청 URL에서 호스트와 프로토콜을 제거하고, 현재 페이지 주소를 기반으로 요청을 보내도록 수정합니다.
예를 들어, fetch 요청 시 http://localhost:8000 대신 상대경로('/api/...')로 요청을 보내도록 변경합니다.
6. 최종 설정
CORS 설정:

CORS_ALLOWED_ORIGINS: http://mydj.com:3000 추가
CORS_ALLOW_CREDENTIALS = True: 쿠키 자동 전송 허용
Session Cookie 설정:

SESSION_COOKIE_DOMAIN = ".mydj.com": 루트 도메인 하위의 모든 서브도메인 간 쿠키 공유
브라우저에서 요청:

fetch API의 credentials: 'include' 설정을 추가하여, Session ID 쿠키를 자동으로 요청에 포함시킵니다.
이를 통해 CORS 오류 없이, 세션 인증이 처리되고, 200 응답을 받습니다.
7. 결과
이제 **http://mydj.com:3000**에서 요청을 보내면, **http://api.mydj.com:8000**로의 요청이 정상적으로 처리되며, CORS 오류가 발생하지 않습니다.
또한, Session ID 쿠키가 자동으로 전송되어, 세션 인증이 정상적으로 이루어집니다.
요약:
로컬 개발 환경에서 도메인을 설정하려면 호스트 파일을 수정하여, 도메인을 로컬 IP에 매핑합니다.
Django 서버에서 CORS와 세션 쿠키 설정을 통해 서브도메인 간 쿠키 공유와 CORS 허용을 설정해야 합니다.
프론트엔드에서 **credentials: 'include'**를 설정하여 쿠키를 자동으로 전송하고, 세션 인증을 성공적으로 처리할 수 있습니다.
