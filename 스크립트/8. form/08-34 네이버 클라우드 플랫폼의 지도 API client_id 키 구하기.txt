Naver Cloud Platform API를 사용하는 지도 위젯을 만들어 볼 텐데요.
그에 앞서 네이버 클라우드 플랫폼의 지도 Client ID를 먼저 얻어야 합니다.
ncloud.com 사이트에서 회원가입을 먼저 진행해 주셔야 하고요.
Ncloud에서는 동일한 전화번호 혹은 동일한 사업자 번호로 여러 계정을 만들 수 있는데,
지도 API 최초 등록일이 가장 빠른 계정이 대표 계정이 된다고 합니다.
대표 계정 한 개에 한해서만 지도 API 무료 이용량이 제공된다고 하니 로그인 후에 반드시
대표 계정 확인 메뉴를 통해 대표 계정 여부를 확인해 주세요
대표 계정이 아니시다면 대표 계정으로 꼭 로그인 후에 애플리케이션을 생성해 주시기 바랍니다
슬라이드 하단에 안내된 NCP 애플리케이션 목록 페이지에 접속하시면 이와 같이 AI Naver
API의 애플리케이션 페이지가 뜹니다.
NCP 서비스 업데이트에 따라 페이지 UI 구성이 다소 달라질 수도 있습니다.
새 애플리케이션 등록 버튼을 클릭하면 이처럼 애플리케이션 생성 폼이 뜹니다.
애플리케이션 이름으로 Naver Map Widget DV를 입력하겠고요.
실서비스 용으로는 Naver Map Widget PROD 이름으로 별도 애플리케이션으로 생성하셔
관리하시길 추천드립니다.
사용할 지도 API는 웹브라우저의 JavaScript 지도 API에서 사용할 수 있는
4개의 API, Web Dynamic Map, Static Map, Geocoding,
Reverse Geocoding을 선택해 줍니다.
Mobile Dynamic Map은 Android, iOS 네이티브 앱에서 사용하는
지도이기에 선택하지 않았습니다.
Web Dynamic Map은 웹브라우저를 통해 동쪽으로 지도를 탐색하고 줌인아웃이 가능한
지도를 제공합니다.
사용자의 상호작용에 따라 실시간으로 지도가 업데이트됩니다.
특정 장소를 검색하거나 지도 위에 마크를 추가하는 등의 작업이 가능합니다.
스태틱 맵은 단일 이미지 지도입니다.
이 이미지는 동적으로 변경되지 않습니다.
위치가 변경되면 우리가 매번 새 이미지를 요청해야 합니다.
자바스크립트 구동이 어렵거나 회사 위치 안내처럼 사용자 상호작용 없이 지도를 보여주기만
하는 경우에 유용합니다.
지오코딩은 주소를 위도와 경도의 좌표로 변환하는 기능을 말합니다.
위치 검색 기능으로 활용할 수 있는데요.
서울특별시 강남구와 같이 주소를 통해 해당 위치의 위도와 경도를 조회할 수 있습니다.
리버스 지오코딩은 이와 반대로 위도와 경도 좌표를 가지고 그 위치의 주소나 지명을 알아내는
기능입니다. 서비스 환경 등록에서 웹서비스 URL로 http://localhost:8000
주소를 입력하고요.
여기에 등록된 주소가 아닌 주소의 웹페이지에서 지도 API를 호출하면 NCP에서 해당
요청을 거부합니다.
서비스 주소가 여럿이라면 여럿 입력하실 수도 있습니다.
등록 버튼을 클릭하면 다시 애플리케이션 목록 페이지로 넘어오시고요.
방금 생성한 애플리케이션을 확인하실 수 있습니다.
당일, 당월 사용량도 확인하실 수 있고, 사용량에 대한 한도 설정 및 알림도 설정하실 수
있습니다. 인증 정보 버튼을 클릭하시면 그 애플리케이션의 클라이언트 ID와 클라이언트
시크릿 키를 확인하실 수 있습니다.
지도에서는 클라이언트 ID 값만 필요하니 클라이언트 ID 값만 메모장에 복사해주세요.
애플리케이션 목록 페이지에서 대표 계정 확인 버튼을 클릭하여 대표 계정 여부를 꼭 확인해
주세요 애플리케이션
등록 버튼을 클릭하시고요
개발용 애플리케이션 이름으로 Naver Map Widget Dev를 지정했고요 사용할
API로서 웹 다이나믹 맵, 스태틱 맵, 지오 코딩, 리버스 지오 코딩을 선택해 줍니다
웹 서비스 URL로는 http://localhost:8000 주소를 입력하고 꼭 추가 버튼을
클릭해 주세요
그리고 등록 버튼을 클릭하시면 애플리케이션이 생성이 됩니다.
인증 정보 버튼을 클릭하여 클라이언트 아이디 값을 복사합니다.
